# 📝 プロジェクト仕様書：Desknets Login Module

## 概要
デスクネッツ（Desknets NEO）への自動ログイン、およびウェブメールの更新（受信）を担当する共通ユーティリティ部品。将来的な複数ツールでの認証処理の共通化を目的とする。

## 📁 構成スクリプトとその役割

本モジュールは、手動では手間のかかる「ログイン」と「メール受信のトリガー」という2つの定型操作を自動化します。

- **`login.py` (Main Module)**:
    - **役割**: 指定されたID/PWを用いてデスクネッツへログインし、「ログイン済みブラウザ状態（WebDriver）」を後続の処理へ引き渡す。
    - **特徴**: 認証の入り口。他ツール（検査成績書取得等）から呼び出されることを前提としている。

- **`navigate_webmail_and_update.py` (Helper Module)**:
    - **役割**: ログイン済みの状態から「ウェブメール」画面へ遷移し、更新（受信）ボタンを強制的にクリックしてメールリストを最新にする。
    - **理由**: 二要素認証コードがメールで届く際、まずデスクネッツ側で「メールを受信した状態」を確実に作るための前処理として存在する。
    - **注意**: 本スクリプトはメール内容の抽出は行わず、「画面を最新にする」操作に特化している。

## ① 要件定義

### 1-A. 要求分析
- **ユーザー要求**: デスクネッツを通じた認証プロセスの自動化、およびメール受信の強制更新。
- **ビジネス目的**: 開発の練習、および複数ツール（検査成績書、SDS等）での認証処理の再利用性の確保。

### 1-B. 要件策定
- **入力**: ログインID、パスワード、組織ID（.env経由）。
- **出力**: ログイン済み、またはメール受信更新済みのブラウザドライバ (Selenium WebDriver)。

### 1-C. デプロイメント戦略
- **提供形式**: Route C (ライブラリ型/共通部品)

## ② 利用手順 (Usage)

### 1. セットアップ
プロジェクトルートにある `.env` ファイルに、以下のログイン情報を設定してください。
```text
DESKNETS_LOGIN_URL="https://www.example.com/login"
DESKNETS_ORG_ID="12345"
DESKNETS_NAME_VALUE="1234"
DESKNETS_PASSWORD="password"
```

### 2. 開発への組み込み
他の Python スクリプトから以下のように呼び出します：
```python
from common_utils.desknets.login import login_and_get_driver

# ログイン済み driver を取得
driver = login_and_get_driver()
```

### 3. 単体テスト実行
`login.py` を直接実行することで、ログイン処理が正常に動作するかを確認できます。

## 💡 技術的メモ
- 他のツールから呼び出される際、プロジェクトルートからの **絶対インポート** (`from common_utils.desknets.login import ...`) を使用すること。
- エラー発生時は `GEMINI.md` に従い、コンソールが即座に閉じないよう待機処理を実装する。
