# Automation Tools for NCI

このリポジトリは、自動ログインツールなどの業務効率化ツールを管理・ビルドするためのプロジェクトです。

---

## � 利用者向け：ツールのダウンロードと実行方法

Windows環境でツールを使用する方は、以下の手順に従ってください。

### 1. 実行ファイル（EXE）の入手
1. GitHub リポジトリ上部の **[Actions]** タブをクリックします。
2. リストの一番上にある、緑色のチェックマーク（✅）が付いた項目をクリックします。
3. 画面下部の "Artifacts" セクションにある **`desknets_login_exe`** をクリックしてダウンロードします。
4. ダウンロードした ZIP ファイルを右クリックして「すべて展開（解凍）」してください。

### 2. フォルダ構成と配置
ツールを正常に動かすには、以下の階層構造でファイルを配置する必要があります。
**重要：実行ファイル本体が含まれるフォルダの「一つ上の階層」に `.env` ファイルを置いてください。**

```text
作業フォルダ（名前は自由）/
├── .env                  (設定ファイル：IDやパスワードを記述)
└── app/                  (フォルダ：名前は自由)
    └── login.exe         (本体：ダウンロードして解凍したもの)
```

### 3. 設定ファイル（.env）の準備
`.env` ファイルを新規作成し、以下の項目を環境に合わせて書き換えて保存してください。
※ `""`（ダブルクォーテーション）などは不要です。

| 項目名 | 設定する内容 | 例 |
| :--- | :--- | :--- |
| **DESKNETS_LOGIN_URL** | Desknet's NEO の**ログイン画面のURL** | `https://example.com` |
| **DESKNETS_ORG_ID** | 選択する**組織（部署）の内部ID**（数字） | `1234` |
| **DESKNETS_NAME_VALUE** | ログインする**氏名の選択値**（通常は氏名コード） | `1234` |
| **DESKNETS_PASSWORD** | あなたの**ログインパスワード** | `password1234` |

#### 💡 DESKNETS_ORG_ID の調べ方
組織選択の画面に表示される部署には、それぞれシステム上の「内部番号」が振られています。
例：
- 生産推進部 → `110`
- 製造部 → `9`

※正しい番号が不明な場合は、管理者または開発担当者へお問い合わせください。

```text
# .env の記述例
DESKNETS_LOGIN_URL=hhttps://example.com
DESKNETS_ORG_ID=1234
DESKNETS_NAME_VALUE=1234
DESKNETS_PASSWORD=password1234
```

### 4. 実行
`app/` フォルダ内の **`login.exe`** をダブルクリックして起動します。

- **初回起動時**: 「Windows によって PC が保護されました」という青い画面が出た場合は、[詳細情報] をクリックしてから [実行] を押してください。
- **要件**: PC に Google Chrome ブラウザがインストールされている必要があります。
- **デバッグ**: 実行中に問題が発生した場合、ウィンドウが閉じずにエラー内容を表示したまま待機します。その画面を開発者に共有してください。

### 💡 トラブルシューティング
実行中にエラーが発生し、ウィンドウが閉じずに止まった場合は以下を確認してください。

| 現象 | 確認ポイント |
| :--- | :--- |
| **.env NOT found...** | `.env` ファイルが `login.exe` が入っているフォルダの**一つ上の階層**にありますか？ |
| **WebDriver Error...** | PC に **Google Chrome** がインストールされていますか？ |
| **ログインテストに失敗...** | `.env` 内の URL、ID、パスワードに打ち間違いはありませんか？ |

---

> [!IMPORTANT]
> **セキュリティに関する注意**
> `.env` ファイルには重要なログイン情報が含まれています。このファイルをメールに添付して送信したり、不特定多数が閲覧できる場所に公開したりしないでください。

## 🛠 開発者向けセットアップ (Mac)

### 依存ライブラリのインストール
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 自動ビルドの仕組み
`common_utils/desknets/login.py` を更新して GitHub へ push すると、GitHub Actions により Windows 用の `.exe` が自動的に作成されます。
PyInstaller のビルド設定は `.github/workflows/build_desknets_login.yml` に記述されています。
